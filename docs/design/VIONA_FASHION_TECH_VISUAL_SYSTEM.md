# VIONA Fashion-Tech Visual System

## Visual principles
- Keep Home in a deep ink, high-contrast shell with readable hierarchy.
- Use champagne gold for hierarchy, not for dense body text.
- Use emerald, cyan, violet, and magenta as controlled accents for actions and status.
- Prefer glass and elevated dark surfaces over white SaaS cards.
- Keep glow soft and cinematic; avoid heavy gaming look.

## Color roles
- Canvas: obsidian/midnight base for Home shell and command areas.
- Primary text: bright neutral for accessibility on dark surfaces.
- Secondary text: softened neutral for descriptions and labels.
- Gold accents: shell hierarchy, emphasis, premium edge highlights.
- Functional accents:
  - Emerald: safe/supportive actions.
  - Cyan: realtime/clock/navigation assist.
  - Violet: insight/demo/reference surfaces.
  - Magenta: care/impact emphasis only.

## Surfaces
- `surface`: default dark panel for contained content.
- `surfaceElevated`: slightly raised dark card for tiles.
- `surfaceGlass`: translucent panel for quick actions and support strips.
- `borderGold`: high-value, low-density framing.
- `borderSubtle`: neutral luminous edge for most cards/panels.
- `shadowPanel` + `glowSoft`: depth and focus without hard neon bloom.

## Cards and tiles
- World pillars remain primary and visually dominant.
- Utility tiles (QR pay, dual clock, index) stay dark, compact, and informative.
- Briefing cards use the same panel language to avoid style drift.
- Care impact strip remains secondary to the four primary worlds.

## Pills and status labels
- Quick action pills: rounded, thin luminous border, icon aura, high legibility.
- Status pills:
  - Lite: cyan
  - Demo: violet
  - Pilot: emerald
  - Coming soon: muted gold
- Preserve safe wording and avoid over-claiming production capability.

## Safety wording guardrails
- Never imply emergency dispatch or authority replacement.
- Keep Safety Assist labels clear and non-claiming.
- Keep all pilot/demo/lite indicators explicit where applicable.

## Content guardrails
- No fake metrics or fabricated live numbers.
- No unlicensed stock imagery.
- Hero visual can remain abstract but must feel premium and intentional.

## Navigation principles (desktop vs mobile)
- Desktop Home uses a single command shell at top.
- No floating account/language/SOS chrome on desktop Home shell.
- No bottom desktop tab overlay on desktop Home shell.
- Mobile and non-Home contexts keep existing navigation behavior.
