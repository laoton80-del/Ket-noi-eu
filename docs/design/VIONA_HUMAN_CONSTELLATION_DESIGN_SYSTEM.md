# VIONA Human Constellation Design System

**Document type:** App-wide visual doctrine  
**Audience:** Product, design, engineering, and AI agents implementing VIONA surfaces  
**Status:** HC.DOC.1 — design doctrine (documentation only)  
**Relationship:** Complements `VIONA_OPERATING_PROTOCOL.md`, `VIONA_GLOBAL_EXPERIENCE_MANIFESTO.md`, and `VIONA_FASHION_TECH_VISUAL_SYSTEM.md`. When visual polish conflicts with signed production truth, **operating protocol and blueprint win**.

---

## 1. Purpose

This document defines VIONA’s **Human Constellation** visual direction for the **entire app**, not a single screen or universe.

VIONA is a **global Vietnamese companion OS**: a human service constellation that connects life, work, learning, travel, safety, and commerce across borders. The interface should feel like a **premium AI and business automation layer** wrapped in a **trusted cross-border life assistant**—warm, cinematic, and calm—not a cold control panel or a vertical demo.

Human Constellation means:

- **People and services** are the story; technology is the connective tissue.
- **Routes, arcs, and nodes** suggest relationship and reach without rigid dashboards or fake “network operations” theater.
- **Maturity is honest**; beauty must not imply production behavior that does not exist.

This doctrine governs **visual language, component behavior, copy posture, and rollout discipline**. It does **not** authorize payment, auth, routing, AI runtime, or database changes by itself.

---

## 2. What VIONA should not look like

Reject patterns that mis-position the product or erode trust:

| Anti-pattern | Why it fails |
|--------------|--------------|
| **Admin dashboard** | VIONA is a companion OS, not an internal ops console for end users. |
| **Crypto wallet** | VIO credits and wallet surfaces are in-app utility, not speculative finance. |
| **Booking-only app** | Commerce and booking are mini-apps inside a broader life graph. |
| **Travel-only app** | Travel is one constellation arm, not the whole product story. |
| **AI demo app** | AI is embedded assistance with honest pilot/demo labeling. |
| **Cyberpunk / gaming UI** | Thick neon, rainbow edges, and arcade glow read as entertainment, not trusted companion software. |
| **Generic SaaS dark mode** | Flat gray cards, stock dashboard grids, and anonymous “enterprise” chrome lack human warmth and Vietnamese-global identity. |

If a surface could belong to any anonymous B2B product, it is **not** Human Constellation.

## 3. Core visual language

### 3.1 Base atmosphere

- **Dark cinematic premium base** — deep navy / ink canvas with controlled elevation, not pure black void or muddy gray wash.
- **Human-global warmth** — gold and soft bloom carry companion feeling; cold blue alone reads as infrastructure, not care.
- **Soft bloom and depth** — offset shadows, satin highlights, and thin glass veils; avoid large radial glow blobs.

### 3.2 Constellation motif

- **Warm gold constellation arcs** — curved luminous trails that suggest routes between people, places, and services.
- **Cyan / blue service intelligence** — focus, language, travel context, and AI assistance accents.
- **Emerald trust nodes** — identity, verification, profile completion, and safety-ready states.
- **Magenta / red only for SOS, demo, or urgent** — never as default decoration.

### 3.3 Hard bans

- No **rigid network grid** as primary decoration.
- No **large abstract orbs** or circular washes behind content.
- No **rainbow** or multi-color edge treatments.
- No **thick neon borders** or cyberpunk frame overload.
- No **one card, many accents** — each component has **one primary accent** unless a documented exception (e.g. SOS).

Motifs **support content**; they do not compete with forms, CTAs, or safety messaging.

## 4. Color semantics

Use color as **meaning**, not decoration.

### Gold

- Brand and premium hierarchy  
- Primary CTA emphasis  
- VIO display and commerce anchor (without implying cash or withdrawable money)  
- Champagne lines, constellation arcs, and command-rail framing  

### Cyan / Blue

- AI assistance and intelligence surfaces  
- Global and cross-border context  
- Travel and movement  
- Language and Smart Trio clarity  
- Focus states and secondary glass actions  

### Emerald

- Identity and profile trust  
- Verified or completion-positive states  
- Local life and community-safe emphasis  
- Safety-ready **guidance** (not live emergency resolution claims)  

### Magenta / Red

- **SOS** and lifeline seriousness  
- Urgent attention within policy  
- Demo / special surfaces only when explicitly labeled  
- Never the default accent for routine commerce or navigation  

### Violet

- **Academy** and learning  
- Creative AI and practice modes  
- Soft, limited use — not a second brand color

## 5. Screen-level application

Apply the same constellation grammar with **universe-appropriate density**. Hero and licensed photography rules from existing packs still apply where documented.

### Home / Hub

- Command rail: brand, greeting/time, language, VIO credits, SOS, account in **one chip system**.  
- Living hero: **preserve licensed hero and dynamic hero imagery**; improve frame, left readability veil, gold accent, subtle cyan edge, CTA hierarchy, and honest status chips.  
- Mini-app cards: Local, Travel, Academy, Business as **connected service nodes** with one accent each.  
- Quick actions: compact “VIONA for you” strip; no fake staffing or 24/7 claims.

### Login / Welcome

- Gateway into the global ecosystem: constellation field behind brand, not a sparse admin form.  
- SOS magenta remains **limited and serious**.  
- Demo entry visible but not dominant.

### SetupProfile / Identity

- Structured identity personalization: faint mapping field, trust spine, emerald/cyan nodes — not a plain admin form.  
- Form rows stay clean; decoration stays behind content.

### Account

- Service graph / account intelligence control center: thin connection language behind cards, gold anchor near profile/VIO areas.  
- No large circular blobs; wallet and checkout behavior unchanged by visual packs.

### SOS

- **Serious lifeline** visual lane: magenta/red discipline, no gamification, no fake dispatch or “active protection” theater.  
- Copy and interaction truth follow SOS product specs and operating protocol.

### Local

- Emerald-led constellation: community, services, daily life.  
- **Lite** or **Pilot** labels when surfaces are preview or gated.

### Travel

- Cyan-led constellation: direction, movement, return, safety-aware guidance.  
- **Lite** until production readiness is signed.

### Academy

- Violet-soft constellation: family learning and AI practice.  
- **Demo** where automation or curriculum is not production-backed.

### Merchant / B2B / AI Receptionist

- Gold commerce anchor with cyan operational intelligence.  
- Pilot and gated labels on automation, payments, and lead flows.  
- No fake booking, payment, or inventory outcomes.

### B2B Wholesale / E-shop Import

- Structured catalog and trade graph — not a fake marketplace.  
- Supplier, SKU, margin, and compliance truth in copy and status chips.

### Admin / Ops

- May use denser tables and diagnostics, but **must not leak** admin-dashboard aesthetics into consumer Home or auth shells.  
- Internal tools follow the same honesty rules for maturity labels.

## 6. Component doctrine

### Buttons

- **Primary:** gold, premium, readable on dark; one primary per view region.  
- **Secondary:** glass / cyan subtle border; no neon frame.  
- **Destructive / SOS:** red/magenta only where policy allows.  
- Disabled states stay legible; do not fake enabled production paths.

### Cards

- Dark glass or elevated ink surface; thin border; inner top highlight optional.  
- **One accent per card** (rail, icon halo, or bottom hairline — not all at once).  
- Image cards: licensed art only; left or bottom scrim for text; no stock-ad collage look.

### Chips

- Shared geometry across command rail, filters, and quick actions.  
- Thin gold or cyan border; soft fill; SOS chip visually distinct.

### Status badges

- Honest maturity: **Lite**, **Demo**, **Pilot**, **Gated**, **Coming soon**, **Frozen**, **Needs review**.  
- Tone mapping: cyan (lite/global), emerald (pilot/trust where appropriate), violet (demo/academy), gold (coming soon / muted), red (urgent only).  
- Avoid **Active** unless production-signed.

### Backgrounds

- Constellation fields: arcs, small nodes, curved trails — **no rigid grid**, **no large orbs**.  
- Fallback dark canvas always present for readability.  
- Overlay scrim above imagery when text sits on photography or generated fields.

### Icons

- Single-accent halo or thin ring; offset depth preferred over radial glow.  
- SOS iconography serious, not playful.

### Image cards

- Do not replace licensed heroes without an approved asset pack.  
- Overlays and frames may change; **pixel content of approved heroes does not** in HC.HOME.1-class packs.

### Command rails

- Integrated greeting, time, and utilities — not a detached dashboard header.  
- VIONA wordmark strong but not oversized.

### Forms

- Clean rows; identity mapping stays in the backdrop layer.  
- Focus markers: small cyan or emerald node — not full-row neon.

### Modals

- Dark glass sheet; thin border; one accent.  
- Safety and payment modals use copy discipline from Section 8; no visual pack changes settlement logic.

## 7. Production safety copy rules

Visual packs **must not** introduce or reinforce unsafe claims unless **production readiness is signed** and documented.

### Banned or restricted claims (unless backed)

- **Active** (as in “feature is live everywhere”)  
- **Live** (staffed support, emergency resolution, or AI always on)  
- **24/7 AI support**  
- **180+ countries active** (or similar scale claims)  
- **Safety Assist Active** (when automation or dispatch is not production-ready)  
- **Fake social proof** (avatar stacks, user counts, “thousands online”)  
- **Payment success** or confirmed paid state without server truth  
- **AI booked / paid / refunded** without governed rails  

### Safer labels (preferred)

- Global Lite preview  
- Safety guide ready  
- AI pilot surface  
- Smart Trio ready  
- Lite  
- Demo  
- Pilot  
- Gated  
- Coming soon  
- Frozen  
- Needs review  

Copy and i18n changes in visual packs stay **Class A** only when they do not change routing, payment, or auth semantics. Ambiguous commercial claims escalate per operating protocol.

## 8. Rollout plan (implementation packs)

Human Constellation ships in **small, reviewable packs**. Each pack has a narrow file allow-list, screenshot evidence, `typecheck` / `lint`, and **no commit without user approval**.

| Pack | Scope |
|------|--------|
| **HC.DOC.1** | This design doctrine (documentation only). |
| **HC.TOKENS.1** | Visual tokens and shared constants (design system layer only). |
| **HC.HOME.1** | Home shell alignment; **preserve hero and dynamic hero images**. |
| **HC.AUTH.1** | Login / Welcome alignment. |
| **HC.IDENTITY.1** | SetupProfile / Account alignment. |
| **HC.LOCAL.1** | Local service constellation surfaces. |
| **HC.TRAVEL.1** | Travel constellation surfaces. |
| **HC.ACADEMY.1** | Academy learning constellation surfaces. |
| **HC.MERCHANT.1** | Merchant / B2B ops constellation surfaces. |
| **HC.SOS.1** | SOS serious lifeline visual alignment (no SOS logic changes in visual-only packs). |

Packs may run in parallel only when file allow-lists do not overlap and reviewers agree on screenshot baselines.

---

## 9. Non-negotiable

- **No broad app rewrite** in a single change.  
- **No all-at-once redesign** across universes.  
- Each pack must be **small, reviewable, and screenshot-approved**.  
- **No commit without user approval** for visual rollout work.  
- **No fake production behavior** in UI or copy.  
- Visual packs **do not** change payment, auth, database, AI runtime, wallet math, checkout, or SOS logic unless a separate signed engineering change explicitly covers that scope.

---

## 10. Review checklist (per pack)

Before marking a visual pack complete:

1. Render path confirmed for the target route(s).  
2. Hero and licensed imagery unchanged if the pack says preserve.  
3. One primary accent per card; no banned motifs (orbs, rainbow, thick neon).  
4. Status chips and hero/support copy pass Section 7.  
5. `npm run typecheck` and `npm run lint` pass.  
6. Desktop (and mobile where relevant) screenshots captured.  
7. `git status` clean of unintended files; **nothing staged** until user approves commit.
