# VIONA Global Experience Manifesto

## 1. Product Feeling
VIONA must feel:
- clear
- premium
- warm
- global
- trustworthy
- calm but powerful
- human + AI, not AI-only

VIONA must not feel:
- dashboard
- admin panel
- crypto app
- charity app
- travel-only app
- booking-only app
- prototype
- template
- floating mascot app
- restaurant/chef-style UI

## 2. First Impression Standard
Within 3 seconds, user must understand:
- VIONA is a multiverse companion OS
- Local / Travel / Academy are the main spaces
- AI and Safety support exist but do not fake production actions
- user can choose a path immediately

## 3. Visual Direction
Define:
- Global Companion Minimalism
- light-first surfaces
- strong typography
- subtle gradients
- minimal decoration
- no random floating elements
- strong but controlled accent colors

## 4. Color Direction
Propose final palette:
- ink: primary text, key icon strokes, high-trust UI anchors
- cloud: app background and low-contrast section containers
- white: card surfaces and modal bodies
- blue: primary actions, active navigation, trustworthy highlights
- indigo: structural shell emphasis and premium depth accents
- teal: Smart Trio / language-context clarity and supportive statuses
- coral: human warmth moments and non-critical emotional accents
- safety red: Safety Assist and critical caution only

## 5. Typography Direction
Define:
- display
- heading
- title
- body
- meta
- caption

## 6. Shell Rules
- compact rail
- no heavy top navy band
- utility dock
- content-first layout
- safe spacing
- desktop and mobile rules

Implementation intent:
- desktop: compressed rail + wide content runway + fixed utility zone
- mobile: first-fold readability + thumb-safe spacing + non-overlapping floating controls

## 7. Home Rules
- Multiverse Hero is primary
- Care Heart Fund is secondary
- Quick actions below hero
- no text wall
- no mixed language
- no fake production claims

## 8. Utility Dock Rules
- Account / Language / VIO belong together
- no floating debug chips
- no clipping
- no hero overlap

## 9. Safety Assist Rules
- SOS must be understandable
- avoid accidental press
- label required
- confirmation before sensitive action
- Lite/Pilot copy required
- no fake emergency dispatch

## 10. Universe Rules
For each:
- Local
- Travel
- Academy
- B2B AI Receptionist

Define:
- emotional goal
- first impression
- primary user action
- safety copy requirement
- forbidden claims

## 11. Card / Surface Rules
Define:
- hero surface
- universe card
- trust card
- impact card
- status pill
- modal
- evidence/admin card

## 12. Motion / Interaction Rules
- subtle
- purposeful
- no gimmick
- no mascot floating shell
- hover/press standards
- modal standards

## 13. Content Rules
- clear first
- short sentences
- no over-explaining above the fold
- Smart Trio consistency
- public brand VIONA / VIO Credits
- no ViGlobal / KNG / VIG Token public UI

## 14. Visual Acceptance Gate
Define checklist:
- Home
- Local
- Travel
- Academy
- B2B
- Admin
- Modal
- Mobile
- Desktop

Each screen must be graded:
- Fail
- Conditional
- Pass
- Pass with delight

Gate policy:
- Any Fail blocks release
- Conditional requires named owner + remediation note
- Pass is minimum for rollout
- Pass with delight is target for flagship surfaces

## 15. Implementation Roadmap
Define packs:
- AE.1 Design System Foundation
- AE.2 Home First-Love Redesign
- AE.3 Utility Dock + Safety Assist
- AE.4 Universe Redesign
- AE.5 B2B/Admin Redesign
- AE.6 Responsive QA
